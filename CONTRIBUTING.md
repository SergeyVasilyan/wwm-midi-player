# Contributing Guide

Thank you for considering contributing to the MIDI Player project!

## How to Contribute
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/your-feature`).
3. Make your changes.
4. Test thoroughly.
5. Submit a pull request.

## Development Setup
```bash
pip install -r requirements.txt
```

## Coding Standards

- Use type hints (ruff compatible).
- Follow PEP8 style guidelines.
- Include docstrings for all public methods.

## Pull Request Process

- Ensure your code builds and runs locally.
- Update documentation if needed.
- One feature or fix per pull request.

## Community

Be respectful, collaborative, and constructive. We welcome bug reports, feature requests, and UI/UX improvements.
